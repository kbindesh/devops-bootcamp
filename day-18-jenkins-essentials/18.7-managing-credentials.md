# Managing Credentials in Jenkins

## Jenkins Credentials - Overview

- Jenkins credentials provide a secure way to store and manage sensitive information like:
  - passwords
  - API tokens
  - SSH keys
  - certificates

- You can store credentials securely in the Jenkins Credentials Manager and then consume it without hardcoding them into scripts or pipelines.
- They are stored in an encrypted format on the Jenkins controller and are typically managed via the _Credentials Plugin_.

## Common Jenkins supported `Credential types`

- Jenkins supports several built-in credential `kinds` depending on the external service you are connecting to:
  1. **Username and Password**
     - Standard login credentials; can be handled as separate components or a username:password string.

  2. **Secret Text**
     - Single token, such as a GitHub Personal Access Token or an API key.
  3. **SSH Username with Private Key**
     - An SSH public/private key pair used for secure git operations or server access.
  4. **Secret File**
     - Secure content stored as a file, which Jenkins temporarily provides to the build environment.
  5. **Certificate**
     - A PKCS#12 certificate file, often with an optional password

## Jenkins Credentials `Scope`

- The `Scope` determines where in Jenkins a credential can be accessed.

- Scope is catagorized as follows:
  1. **Global**
     - Accessible to all jobs, items, and descendant objects on the entire Jenkins controller.

  2. **System**
     - Restricted to the Jenkins controller itself for administrative functions
     - e.g., email authentication or agent connections
  3. **Folder**
     - Credentials restricted to a specific folder and its sub-items, preventing other teams from seeing or using them

## Lab: How to Add a Credentials

- Navigate to Jenkins Dashboard >> Manage Jenkins > Credentials.

- Select the _(global) domain_ under the _System_ store (or a specific folder store).
- Click Add **Credentials** on the left menu.
- Select the _Kind_ (e.g., Secret text) and enter a unique ID. This ID is what you will reference in your Jenkins Job or Jenkinsfile
