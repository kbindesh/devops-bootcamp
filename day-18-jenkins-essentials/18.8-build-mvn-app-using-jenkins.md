# `Practice Lab-01` - Create a Jenkins based CI Pipeline for building a Apache Maven Application

- In this lab, you will learn how to configure and automatically build Maven application on Jenkins server.
- Following topics are covered:
  - Develop a sample Maven App
  - Check-in the Maven App source code into GitHub
  - Configure Jenkins server for building Maven App
  - Create a GitHub Webhook for Jenkins server
  - Create & run a Jenkins Maven Job

## Prerequisites

- A _Jenkins_ server with admin privileges
- A _GitHub_ account

## Step-01: Develop a simple Maven Application

- Launch VS Code (IDE) >> Create a new project folder say **my-app** and open it in the VS Code >> **View** menu >> **Terminal**

  ```
  # Run the following maven command to create a new sample maven project
  mvn archetype:generate "-DgroupId=com.mycompany.app" "-DartifactId=my-app" "-DarchetypeArtifactId=maven-archetype-quickstart" "-DarchetypeVersion=1.5" "-DinteractiveMode=false"
  ```

- Get inside the project folder:

  ```
  cd my-app
  ```

  - The **src/main/java** directory contains the project source code
  - The **src/test/java** directory contains the test source
  - The **pom.xml** file is the project's Project Object Model, or POM

## Step-02: Compile & Validate the App (locally)

```
# Validate Maven project
mvn validate

# Compile Maven project
mvn compile

# To see the output of your Maven app, run the compiled version of app (.class) present in target folder

java '-XX:+ShowCodeDetailsInExceptionMessages' '-cp' 'target\classes' 'com.mycompany.app.App'

[The preceding command should display the "Hello World" message without any errors]
```

## Step-03: Create a new GitHub repository for Maven App

- Navigate to https://gitHub.com/ >> Sign-in
- Select **Repositories** tab >> click **New** button
  - **Repository Name**: mvn-sample-project
  - **Description**
  - **Scope**: Private

- Click **Create repository** button

## Step-04: Push the Maven App code to GitHub

### Step-4.1: Commit the changes and set the GitHub remote origin

- Launch **VS Code** >> Open your maven project >> **View** menu -> **Terminal**

```
# Get inside the project folder path
cd <your_project_folder_name>

# Initialize the project folder as a git repo
git init

# Stage the changes
git add .

# Commit the changes on default (master) branch
git commit -m "Maven App version 1.0"

# Set the remote Github repo origin | replace repo link with yours
git remote add origin https://github.com/kbindesh/mvn-sample-project.git

# (Optional) Rename a branch
git branch -M main
```

- **Push the changes to Github (remote origin)**

```
git push -u <remote_origin_alias> <branch_name>

# Example
git push -u origin main
```

- If prompted, enter the username and password of your GitHub account.

- Verify the application code from your GitHub repository.

## Step-05: Configure Jenkins Server for building Maven Application

### Step-5.1: Install & Configure `Java` (prerequisite for Maven)

- Java official download page - https://www.oracle.com/in/java/technologies/downloads/

- _INFO_: You will already have Java running on your Jenkins server as it is also a prerequiste for setting-up Jenkins.

```
# To check if Java is present on your Jenkins server
java --version

[If the preceding command display installed version of Java, you can skip this entire step else continue]

# Install Java 21
sudo dnf install -y java-21-amazon-corretto-devel

# To verify java installation
java --version
```

### Step-5.2: `Install and Configure Maven` on Jenkins server

- Apache Maven official download page: https://maven.apache.org/download.cgi

```
# Download the Apache Maven package
wget https://dlcdn.apache.org/maven/maven-3/3.9.14/binaries/apache-maven-3.9.14-bin.tar.gz

# Extract the downloaded Maven package
sudo tar xvf apache-maven-3.9.14-bin.tar.gz -C /opt

# Create a symlink to the maven folder
sudo ln -s /opt/apache-maven-3.9.14 /opt/maven

# Create a script file named "maven.sh" in the /etc/profile.d directory
sudo vi /etc/profile.d/maven.sh

[Add the following to the script and save the file]
export M2_HOME=/opt/maven
export PATH=${M2_HOME}/bin:${PATH}
```

- Now, add execute permission to the `maven.sh` script file:

```
sudo chmod +x /etc/profile.d/maven.sh

# Source the script for changes to take immediate effect
source /etc/profile.d/maven.sh

# Verify Apache Maven installation
mvn --version
```

### Step-5.3: Install `Git`

```
sudo dnf install -y git
```

### Step-5.4: Install `Jenkins Plugins`

- Navigate to the Jenkins Dashboard >> **Manage Jenkins** >> **Plugins** >> Select **Available** tab
- Search following plugins and install it:
  - **Maven Integration**
  - **Github Plugin**

- Once all the required plugins are installed, verify it from the **Installed** tab of `Plugins` manager.

## Step-06: Save the required `Tools location` on Jenkins - Maven and Git

- Navigate to Jenkins server dashboard >> **Manage Jenkins** >> **Tools**

- **Git**
  - Name: Default
  - Path to git executable: /usr/bin/git

- **Maven**
  - Name: Maven-3.9.14
  - MAVEN_HOME: /opt/maven

## Step-07: Generate GitHub Personal Access Token (PAT) and share it with Jenkins server

- In order to let Jenkins securely access your GitHub repo, Jenkins server must authenticate itself.
- In this lab, we will use HTTPS based authentication using `GitHub PAT`.

### Step-7.1: Generate GitHub Personal Access Token (PAT)

- Sign in to your GitHub account.
- Click on your profile picture in the upper-right corner >> **Settings** >> **Developer settings** >> **Personal Access Token** >> **Tokens (classic)**
- Click **Generate new token (classic)**
- Copy the generated token to your clipboard.

**Note**: You won't be able to see this token again, so store it securely somewhere.

### Step-7.2: Save the generated GitHub PAT on Jenkins server

- Navigate to Jenkins Dashboard >> **Manage Jenkins** >> **Credentials**
- Create a new Global credential
  - Type: Username and Password
  - Username: $your-github-username
  - Password: $your-GITHUB-PAT
  - ID: github-credentials
  - Description: Github Credentials

## Step-08: Create a `GitHub Webhook` for Maven Jenkins job

- Switch to your GitHub repository >> select **Settings** tab
- Select Webhooks from the left-side panel >> Add Webhook
  - **Payload URL**: http://$YOUR_JENKINS_SERVER_DNS:8080/github-webhook/
    - Example: http://ec2-98-92-83-225.compute-1.amazonaws.com:8080/github-webhook/
  - **Content type**: Application/json
  - **Which events would you like to trigger this webhook?**: Just the push event
  - **Active**: Enabled
  - Click **Add Webhook** button

## Step-09: Create a new Jenkins job

- **Name**: mvn-build-job-01
- **Type**: Freestyle Job
- **Source Code Management**
  - Select _Git_
    - **Repository URL**: <github_repo_url>
    - **Branch to build**: <github_branch_name>
- **Build Trigger**
  - Poll SCM (GitHub Webhook)
- **Build**
  - Build-step type: Invoke Maven top-level targets
    - **Maven Version**: Maven-3.9.14
    - **Goals**: clean install

## Step-10: Trigger the Jenkins Job automatically

- To trigger the jenkins job, push a small change in your GitHub repo as it is configured with the GitHub webhook.

- Now, the source code will be cloned from Github to Jenkins server and will create a build artifact (.jar file) on the following location:

```
/var/lib/jenkins/workspace/<your_jenkins_job_name>/target/
```

- List the _target_ directory of your workspace to see the build artifact (.jar file).

```
# Switch to your workspace directory
cd /var/lib/jenkins/workspace/<your_jenkins_job_name>

# Finally, test the app by executing the app build (.jar)
java -cp target/my-app-1.0-SNAPSHOT.jar com.mycompany.app.App

[The preceding command should display the "Hello World" message without any errors]
```
