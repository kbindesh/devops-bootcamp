# `Practice Lab-01` - Create a Jenkins based CI pipeline for building a Maven Application

- In this lab, you will learn how to configure and automatically build Maven application on Jenkins server.
- Following topics are covered:
  - Develop a sample Maven App
  - Check-in the Maven App source code into GitHub
  - Configure Jenkins server for building Maven App
  - Create a GitHub Webhook for Jenkins server
  - Create & run a Jenkins Maven Job

## Prerequisites

- A _Jenkins_ server with admin privileges
- A _GitHub_ account

## Step-01: Develop a simple Maven Application

- Launch VS Code (IDE) >> Create a new project folder say **my-app** and open it in the VS Code >> **View** menu >> **Terminal**

  ```
  # Run the following maven command to create a new sample maven project
  mvn archetype:generate "-DgroupId=com.mycompany.app" "-DartifactId=my-app" "-DarchetypeArtifactId=maven-archetype-quickstart" "-DarchetypeVersion=1.5" "-DinteractiveMode=false"
  ```

- Get inside the project folder:

  ```
  cd my-app
  ```

  - The **src/main/java** directory contains the project source code
  - The **src/test/java** directory contains the test source
  - The **pom.xml** file is the project's Project Object Model, or POM

## Step-02: Compile & Validate the App (locally)

```
# Validate Maven project
mvn validate

# Compile Maven project
mvn compile

# To see the output of your Maven app, run the compiled version of app (.class) present in target folder

java '-XX:+ShowCodeDetailsInExceptionMessages' '-cp' 'target\classes' 'com.mycompany.app.App'

[The preceding command should display the "Hello World" message without any errors]
```

## Step-04: Push the Maven App code to GitHub

### Step-4.1: GitHub `Username & Password based Authentication` to the push code

- Launch **VS Code** >> Open your maven project >> **View** menu -> **Terminal**

```
# Get inside the project folder path
cd <your_project_folder_name>

# Initialize the project folder as a git repo
git init

# Stage the changes
git add .

# Commit the changes on default (master) branch
git commit -m "Maven App version 1.0"

# Set the remote Github repo origin | replace repo link with yours
git remote add origin https://github.com/kbindesh/mvn-sample-project.git

# (Optional) Rename a branch
git branch -M main
```

- **Push the changes to Github (remote origin)**

```
git push -u <remote_origin_alias> <branch_name>

# Example
git push -u origin main
```

- If prompted, enter the username and password of your GitHub account.

- Verify the application code from GitHub repository.

### Step-4.2: (Alternative approach) GitHub `Token based authentication` to push code to GitHub

#### Generate GitHub Personal Access Token (PAT)

- Sign in to your GitHub account.
- Click on your profile picture in the upper-right corner >> **Settings** >> **Developer settings** >> **Personal Access Token**.
- Click Generate token.
- Copy the token to your clipboard.

**Note**: You won’t be able to see this token again, so store it securely somewhere.

#### Configure git to use your GitHub token

- To authenticate Git operations with your token, you need to update the URL of your GitHub repository to include the token.

```
# Navigate to your project directory
cd <maven_project_folder>

# Update the remote repo (origin) URL
git remote set-url origin https://<TOKEN>@github.com/<USERNAME>/<REPOSITORY>.git

[In the preceding command, replace <TOKEN> with your actual Github token, USERNAME with your GitHub username, and REPOSITORY with the name of your repository.]
```

## Step-05: Configure Jenkins Server for building Maven Application

### Step-5.1: Install & Configure Java (prerequisite for Maven)

- _INFO_: You will already have Java running on your Jenkins server as it is also a prerequiste for setting-up Jenkins.

```
# To check if Java is present on your Jenkins server
java --version

[If the preceding command display installed version of Java, you can skip this entire step else continue]

# Install Java 17
sudo yum install -y java-17-amazon-corretto.x86_64

# To verify java installation
java --version
```

### Step-5.2: Set Path for Java

- Official Java Download link: https://www.oracle.com/in/java/technologies/downloads/

```
find /usr/lib/jvm/java* | head -n 3

[From the preceding command output, copy "/usr/lib/jvm/java-17-amazon-corretto.x86_64" path]

# Edit the bash_profile
vi ~/.bash_profile

# Create a new variable JAVA_HOME
JAVA_HOME=/usr/lib/jvm/java-17-amazon-corretto.x86_64

# Add JAVA_HOME to the existing path
PATH=$PATH:$HOME/bin:$JAVA_HOME

[Save the file and exit]
```

- **Verify the Java path**

```
echo $PATH
[The preceding command will give you the updated PATH]

# In order to refresh the path
source ~/.bash_profile

# Again, display the PATH to get the updated values
echo $PATH

[The preceding command should display java & maven home paths]
```

### Step-5.3: Install `Jenkins Plugins`

- Navigate to the Jenkins Dashboard >> **Manage Jenkins** >> **Plugins** >> Select **Available** tab
- Search following plugins and install it:
  - **Maven Integration**
  - **Maven Invoker**
  - **Github Plugin**

- Once all the required plugins as installed, verify it from the **Installed** tab of plugin manager.

### Step-5.4: `Install and Configure Maven` on Jenkins server

- Apache Maven official download page: https://maven.apache.org/download.cgi

```
# Move to /opt directory
cd /opt

# Download the Maven binary
wget https://dlcdn.apache.org/maven/maven-3/3.9.8/binaries/apache-maven-3.9.8-bin.tar.gz

wget https://dlcdn.apache.org/maven/maven-3/3.9.11/binaries/apache-maven-3.9.11-bin.tar.gz

# Unzip the downloaded maven tarball
tar -xvzf apache-maven-3.9.11-bin.tar.gz

# List all the file to see the unzipped maven directory
ls -l apache-maven-3.9.11

# Get inside the maven home directory
cd apache-maven-3.9.11
```

- **Setup home path for Maven**

```
# Update the bash profile with maven path
vi ~/.bash_profile

# Create M2_HOME and M2 variable with maven location specs
M2_HOME=/opt/apache-maven-3.9.11
M2=/opt/apache-maven-3.9.11/bin

# Update the PATH variable | Add Maven path
PATH=$PATH:$HOME/bin:$JAVA_HOME:$M2_HOME:$M2

[Save the file and exit]

source ~/.bash_profile

# Verify the PATH with java and maven variables
echo $PATH
```

- Now, check if you're able to access Maven:

```
# To verify Maven installation
mvn --version
```

### Step-5.5: Install `Git`

```
sudo yum install -y git
```

## Step-08: Create a new Jenkins job

- **Name**: mvn-build-job-01
- **Type**: Maven project
- **Source Code Management**
  - Select _Git_
    - **Repository URL**: <github_repo_url>
    - **Branch to build**: <github_branch_name>
- **Build**
  - **Maven Version**: Maven-3.9.11
  - **Root POM**: pom.xml
  - **Goals and Options**: clean install

## Step-09: Trigger the Jenkins Job automatically

- To trigger the jenkins job, push a small change in your GitHub repo as it is configured with the GitHub webhook.

- Now, the source code will be cloned from Github to Jenkins server and will create a build artifact (.jar file) on the following location:

```
/var/lib/jenkins/workspace/<your_jenkins_job_name>/target/
```

- List the _target_ directory of your workspace to see the build artifact (.jar file).

```
# Switch to your workspace directory
cd /var/lib/jenkins/workspace/<your_jenkins_job_name>

# Finally, test the app by executing the app build (.jar)
java -cp target/my-app-1.0-SNAPSHOT.jar com.mycompany.app.App

[The preceding command should display the "Hello World" message without any errors]
```
