# Amazon Elastic Compute Cloud (EC2)

## Amazon EC2 - Key Concepts

- Amazon Machine Image (AMI)
- Instance Types
- Key Pair
- Elastic IP
- Security Group (SG) - Inbound and Outbound Rules
- EBS Volume
- EC2 User data and Metadata
- Elastic Load Balancer (ELB)
- Auto Scaling Group (ASG)
- EC2 Instance Pricing

## Common Port Specifications

| Protocol | Port | Description          | Used For           |
| -------- | ---- | -------------------- | ------------------ |
| TCP      | 22   | SSH (Secure Shell)   | Linux Management   |
| TCP      | 3389 | RDP (Remote Desktop) | Windows Management |
| TCP      | 80   | HTTP                 | Web Traffic        |
| TCP      | 443  | HTTPS                | Secure Web Traffic |
| TCP/UDP  | 53   | DNS                  | Domain Name System |

## Lab: Create an Amazon EC2 Instance (Windows)

1. Create a new EC2 Instance with Windows AMI
2. Connect to the EC2 instance over RDP
3. Configure the instance as a webserver
4. Deploy an application
5. Access the application over the internet

## Lab: Create an Amazon EC2 Instance (Linux)

1. Create a new EC2 Instance with Linux AMI
2. Connect to the EC2 instance over SSH
3. Configure the instance as a webserver
4. Deploy an application
5. Access the application over the internet

## Lab: Setting-up Amazon Elastic Load Balancer (ELB) for EC2 instance hosted Application

1. Create 2 EC2 Instances (Linux) in two different AZs.

2. Configure both the EC2 instances as a Webserver (nginx).

3. Create a new Target Group and add both the EC2 instances into it.

4. Create an Application Load Balancer (ALB) and integrate it with the Target Group created in the previous step.
