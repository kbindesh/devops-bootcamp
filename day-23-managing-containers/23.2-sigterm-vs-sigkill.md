# SIGTERM vs SIGKILL

- In Docker, `SIGTERM` and `SIGKILL` are Linux signals used to terminate processes within a container.
- The primary difference is in how the application responds to them.

## SIGTERM (Signal 15)

- A "graceful shutdown" signal.
- It tells the application to stop, allowing it time to save its state, close database connections, and flush buffers.

- Docker command which triggers SIGTERM signal is `docker container stop`

- Docker sends SIGTERM to the main process (PID 1) and waits for a grace period (default 10 seconds).

## SIGKILL (Signal 9)

- A forced, immediate termination.
- The kernel kills the process instantly without notifying it.
- With SIGKILL, the application has no chance to perform any cleanup.

- Docker command which triggers SIGKILL signal is `docker container kill`

- If the application does not exit within the 10-second grace period after issuing a `SIGTERM` signal, Docker sends `SIGKILL` to end it forcefully.
