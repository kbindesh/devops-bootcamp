# STDIN, STDOUT AND STDERR

- In Docker, STDIN, STDOUT, and STDERR are the standard I/O streams used by a container's main process.
- By default, Docker captures STDOUT and STDERR for logging and can attach STDIN for interactive use.

## I/O Streams in Docker

- **STDIN (Standard Input)**
  - Used to send input (like commands) to a container.
  - It is closed by default unless the `-i` or `--interactive` flag is used.

- **STDOUT (Standard Output)**
  - Used for normal process output.
  - Docker captures this stream and makes it available via the `docker logs` command.

- **STDERR (Standard Error)**
  - Used for error messages and diagnostics.
  - Like STDOUT, it is captured by Docker's logging driver.

## Common Docker Flags for Streams

- When you use commands like `docker run` or `docker exec`, you can control how these streams interact with your terminal.
  - `-i` or `--interactive`
    - Keeps STDIN open even if not attached.

  - `-t` `--tty`
    - Allocates a pseudo-TTY, which makes the terminal behave like a real console (enabling colors and proper formatting).
  - `-a` or `--attach`
    - Manually attach to specific streams (e.g., -a stdout -a stderr).
  - `-d` or `--detach`
    - Runs the container in the background without attaching to any streams.
