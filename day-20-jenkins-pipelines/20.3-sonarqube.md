## 02. Lab: Integrate SonarQube with Jenkins pipeline

- In this lab, you will learn how to integrate `SonarQube` with `Jenkins pipeline` to automate the code review process.
- Lab comprises of following components:
  - **Cloud Platform**: Amazon Web Services
  - **Maven**: Project Orchestrator (Java)
  - **GitHub**: SCM
  - **Jenkins**: Automation server
  - **SonarQube**: Static Code Analysis Tool

### Step-2.1: `Prerequisites`

- GitHub Account with a sample Maven App repository
- Jenkins server with the following tools and plugins installed:
  - **Tools**
    - Maven (3.9.13 or above)
    - Git
    - Java (jdk21 or above)
  - **Plugins**
    - Maven Integration
    - Maven Invoker
    - GitHub
    - Pipeline: Stage view

### Step-2.2: Save the required Tools location on Jenkins

- Navigate to Jenkins server dashboard >> **Manage Jenkins** >> **Tools**

- **Git**
  - Name: Default
  - Path to git executable: /usr/bin/git

- **Maven**
  - Name: Maven-3.9.14
  - MAVEN_HOME: /opt/maven

### Step-2.5: Generate SonarQube Authentication Token for Jenkins

### Step-2.7: Setup `SonarScanner` on Jenkins server

- **Refernece**
  - https://docs.sonarsource.com/sonarqube-server/2025.5/analyzing-source-code/scanners/sonarscanner-for-maven
  - https://docs.sonarsource.com/sonarqube-server/2025.5/analyzing-source-code/scanners/jenkins-extension-sonarqube

#### Step-2.7.1: Download and Configure SonarScanner (manually)

- https://docs.sonarsource.com/sonarqube-server/10.8/analyzing-source-code/scanners/sonarscanner

```
cd /opt

sudo wget https://binaries.sonarsource.com/Distribution/sonar-scanner-cli/sonar-scanner-cli-8.0.1.6346-linux-x64.zip

sudo unzip sonar-scanner-cli-8.0.1.6346-linux-x64.zip
sudo mv sonar-scanner-cli-8.0.1.6346-linux-x64/ sonar-scanner/
```

#### Step-2.7.2: Set the SonarScanner location in /etc/profile.d directory

- Create a new file `/etc/profile.d/sonar-scanner.sh` with the following contents and save:

```
export PATH="$PATH:/opt/sonar-scanner/bin"
```

- To take the changes immediate effect, source the created file:

```
source /etc/profile.d/sonar-scanner.sh
```

- Ensure the `sonar-scanner` command is recognized by checking the version

```
sonar-scanner -v
```

### Step-2.9: Save SonarQube Server details on Jenkins server

- Jenkins Dashboard >> **Manage Jenkins** >> **System**.
- Scroll down all the way to **SonarQube server section** (thanks to sonarqube scanner plugin).
- Click on **Add SonarQube** button
  - Name: sonarqube-server
  - Server URL: https://sonarcloud.io
  - **Server Authentication Token**: <select_sonar_token_we_created_earlier>
- Click **Save** button

### Step-2.10: Create a `Jenkinsfile` for the Maven project

- Add SonarQube stage to the Jenkinsfile

```
mvn package sonar:sonar '-Dsonar.projectKey=binorg_mvn-project' '-Dsonar.organization=binorg' '-Dsonar.projectName=mvn-project' '-Dsonar.host.url=https://sonarcloud.io' '-Dsonar.token=1c599b7e55c68165db79fa6e12632a0ab2b44045' -X
```

### Step-2.11: Update pom.xml file for code review using SonarQube

- Reference: https://central.sonatype.com/artifact/org.sonarsource.scanner.maven/sonar-maven-plugin/versions

- Add a _Dependency_ (org.sonarsource.scanner.maven) block for sonar scanner:

```
<dependency>
  <groupId>org.sonarsource.scanner.maven</groupId>
  <artifactId>sonar-maven-plugin</artifactId>
  <version>5.5.0.6356</version>
</dependency>
```

- For a sample pom.xml file, you may refer to [pom.xml](./config-files/pom.xml)

### Step-2.12: Push the changes to GitHub

```
git add .

git push -u origin main
```

### Step-2.13: Create and Configure GitHub Webhook for Jenkins

- Navigate to your GitHub Account >> Select your repository which has the application source code >> Select **Setting** tab
- Select **Webhooks** >> **Add Webhook**
  - **Payload URL**
    - http://<YOUR_JENKINS_SERVER_PUBLIC_IP_OR_DNS>:8080/github-webhook/
    - Example: http://xx.xx.xx.xx:8080/github-webhook/
  - **Content Type**: application/json
  - **SSL Verification**: Enable SSL Verification
  - **Which events would you like to trigger this webhook?**: Just the push event
  - **Active**: Enable
- Click **Add Webhook** button

### Step-2.14: Create a new Jenkins pipeline job

- Navigate to the Jenkins Dashboard >> New Item
  - Job Name: HelloMvnApp-Main-Build
  - Type: Pipeline
  - Trigger type: GitHub (SCM) Webhook
  - Jenkinsfile: Fetch from the Git
    - Git repo URL: https://github.com/kbindesh/maven-project.git
    - branch: main

### Step-2.15: Trigger the Jenkins Job

- To test the webhook tigger, make changes to the app code on your local system and push them to GitHub repository.
- This will activate the webhook, and Jenkins should automatically trigger the job.

### Step-2.16: Verify the Jenkins Build and Code Review results

- For Build result, kindly refer to the Jenkins _Build_ page and _Console Output_ section.

- For Code review results, kindly refer to the _SonarQube Dashboard_.
